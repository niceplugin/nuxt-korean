## [alias](#alias)

JavaScript 및 CSS 내에서 커스텀 디렉토리에 접근할 수 있도록 추가 별칭을 정의하여 DX(개발자 경험)를 향상시킬 수 있습니다.

- **타입**: `object`
- **기본값**

```json
{
  "~": "/<srcDir>",
  "@": "/<srcDir>",
  "~~": "/<rootDir>",
  "@@": "/<rootDir>",
  "#shared": "/<rootDir>/shared",
  "assets": "/<srcDir>/assets",
  "public": "/<srcDir>/public",
  "#build": "/<rootDir>/.nuxt",
  "#internal/nuxt/paths": "/<rootDir>/.nuxt/paths.mjs"
}
```

<callout>

**참고**: webpack 컨텍스트(이미지 소스, CSS - JavaScript는 제외) 내에서는
별칭 앞에 `~`를 붙여서 접근해야 합니다.

</callout>

<callout>

**참고**: 이 별칭들은 자동으로 생성된 `.nuxt/tsconfig.json`에 추가되어 전체 타입 지원과 경로 자동 완성을 제공합니다.
만약 `./.nuxt/tsconfig.json`에서 제공하는 옵션을 더 확장해야 한다면,
여기나 `nuxt.config`의 `typescript.tsConfig` 속성 내에 추가하세요.

</callout>

**예시**:

```js
export default {
  alias: {
    'images': fileURLToPath(new URL('./assets/images', import.meta.url)),
    'style': fileURLToPath(new URL('./assets/style', import.meta.url)),
    'data': fileURLToPath(new URL('./assets/other/data', import.meta.url))
  }
}
```

## [app](#app)

Nuxt 앱 설정.

### [`baseURL`](#baseurl)

Nuxt 애플리케이션의 기본 경로입니다.

예시:

- **타입**: `string`
- **기본값:** `"/"`

**예시**:

```ts
export default defineNuxtConfig({
  app: {
    baseURL: '/prefix/'
  }
})
```

**예시**:

```bash
NUXT_APP_BASE_URL=/prefix/ node .output/server/index.mjs
```

### [`buildAssetsDir`](#buildassetsdir)

빌드된 사이트 에셋의 폴더 이름입니다. `baseURL`(또는 설정된 경우 `cdnURL`)을 기준으로 상대 경로입니다. 이 값은 빌드 시에만 설정되며 런타임에 커스터마이즈해서는 안 됩니다.

- **타입**: `string`
- **기본값:** `"/_nuxt/"`

### [`cdnURL`](#cdnurl)

public 폴더를 제공할 절대 URL(프로덕션 전용)입니다.

예시:

- **타입**: `string`
- **기본값:** `""`

**예시**:

```ts
export default defineNuxtConfig({
  app: {
    cdnURL: 'https://mycdn.org/'
  }
})
```

**예시**:

```bash
NUXT_APP_CDN_URL=https://mycdn.org/ node .output/server/index.mjs
```

### [`head`](#head)

모든 페이지의 `<head>`에 대한 기본 설정을 지정합니다.

- **타입**: `object`
- **기본값**

```json
{
  "meta": [
    {
      "name": "viewport",
      "content": "width=device-width, initial-scale=1"
    },
    {
      "charset": "utf-8"
    }
  ],
  "link": [],
  "style": [],
  "script": [],
  "noscript": []
}
```

**예시**:

```js
app: {
  head: {
    meta: [
      // <meta name="viewport" content="width=device-width, initial-scale=1">
      { name: 'viewport', content: 'width=device-width, initial-scale=1' }
    ],
    script: [
      // <script src="https://myawesome-lib.js"></script>
      { src: 'https://awesome-lib.js' }
    ],
    link: [
      // <link rel="stylesheet" href="https://myawesome-lib.css">
      { rel: 'stylesheet', href: 'https://awesome-lib.css' }
    ],
    // 이 영역은 변경될 가능성이 있습니다
    style: [
      // <style>:root { color: red }</style>
      { textContent: ':root { color: red }' }
    ],
    noscript: [
      // <noscript>JavaScript is required</noscript>
      { textContent: 'JavaScript is required' }
    ]
  }
}
```

## [build](#build)

공유 빌드 설정.

### [`analyze`](#analyze)

Nuxt는 번들을 시각화하고 최적화 방법을 제공합니다.

`true`로 설정하면 번들 분석이 활성화되며, 옵션 객체를 전달할 수도 있습니다: [webpack용 옵션](https://github.com/webpack-contrib/webpack-bundle-analyzer#options-for-plugin) 또는 [vite용 옵션](https://github.com/btd/rollup-plugin-visualizer#options).

- **타입**: `object`
- **기본값**

```json
{
  "template": "treemap",
  "projectRoot": "/<rootDir>",
  "filename": "/<rootDir>/.nuxt/analyze/{name}.html"
}
```

**예시**:

```js
analyze: {
  analyzerMode: 'static'
}
```

### [`templates`](#templates)

이 옵션 대신 `@nuxt/kit`의 `addTemplate` 사용을 권장합니다.

- **타입**: `array`

**예시**:

```js
templates: [
  {
    src: '~/modules/support/plugin.js', // `src`는 절대 또는 상대 경로 모두 가능
    dst: 'support.js', // `dst`는 프로젝트 `.nuxt` 디렉토리 기준 상대 경로
  }
]
```

### [`transpile`](#transpile)

특정 의존성을 Babel로 트랜스파일하려면 여기에 추가할 수 있습니다. transpile의 각 항목은 패키지 이름, 함수, 문자열 또는 의존성 파일 이름과 일치하는 정규식 객체일 수 있습니다.

함수를 사용하여 조건부로 트랜스파일할 수도 있습니다. 함수는 객체({ isDev, isServer, isClient, isModern, isLegacy })를 인자로 받습니다.

- **타입**: `array`

**예시**:

```js
transpile: [({ isLegacy }) => isLegacy && 'ky']
```

## [buildDir](#builddir)

빌드된 Nuxt 파일이 저장될 디렉토리를 정의합니다.

- **타입**: `string`
- **기본값:** `"/<rootDir>/.nuxt"`

**예시**:


## [components](#components)

Nuxt 컴포넌트 자동 등록을 설정합니다.

여기에 설정된 디렉토리의 모든 컴포넌트는 페이지, 레이아웃(및 기타 컴포넌트)에서 명시적으로 import하지 않고도 사용할 수 있습니다.

- **타입**: `object`
- **기본값**

```json
{
  "dirs": [
    {
      "path": "~/components/global",
      "global": true
    },
    "~/components"
  ]
}
```

**참고**: [`components/` 디렉토리 문서](https://nuxt.com/docs/guide/directory-structure/components)

## [css](#css)

글로벌로 설정할 CSS 파일/모듈/라이브러리를 정의할 수 있습니다(모든 페이지에 포함됨).

Nuxt는 확장자를 통해 파일 타입을 자동으로 추측하고 적절한 전처리기를 사용합니다. 필요하다면 해당 로더를 직접 설치해야 합니다.

- **타입**: `array`

**예시**:

```js
css: [
  // Node.js 모듈을 직접 로드(여기서는 Sass 파일)
  'bulma',
  // 프로젝트 내 CSS 파일
  '~/assets/css/main.css',
  // 프로젝트 내 SCSS 파일
  '~/assets/css/main.scss'
]
```

## [experimental](#experimental)

```ts
// 라우트 슬러그에 따라 가져오는 데이터가 달라지지만, Nuxt는 키에 반영되지 않아 이를 알 수 없습니다.
const route = useRoute()
const { data } = await useAsyncData(async () => {
  return await $fetch(`/api/my-page/${route.params.slug}`)
})
// 대신, 가져오는 데이터를 고유하게 식별하는 키를 사용해야 합니다.

  return await $fetch(`/api/my-page/${route.params.slug}`)
})
```

### [`templateImportResolution`](#templateimportresolution)

## [modules](#modules)

각 모듈은 문자열(패키지 이름 또는 파일 경로), 모듈과 옵션 객체를 포함한 튜플, 또는 인라인 모듈 함수일 수 있습니다.
Nuxt는 modules 배열의 각 항목을 node require 경로(`node_modules` 내)로 해석하려 시도한 후, `~` 별칭이 사용된 경우 프로젝트 `srcDir`에서 해석합니다.

- **타입**: `array`

<callout>

**참고**: 모듈은 순차적으로 실행되므로 순서가 중요합니다. 먼저 `nuxt.config.ts`에 정의된 모듈이 로드되고, 그 다음 `modules/`
디렉토리에서 발견된 모듈이 알파벳 순으로 실행됩니다.

</callout>

**예시**:

```js
modules: [
  // 패키지 이름 사용
  '@nuxtjs/axios',
  // 프로젝트 srcDir 기준 상대 경로
  '~/modules/awesome.js',
  // 옵션 제공
  ['@nuxtjs/google-analytics', { ua: 'X1234567' }],
  // 인라인 정의
  function () {}
]
```

```js
  '~/plugins/baz.js', // 클라이언트 & 서버 모두
  { src: '~/plugins/both-sides.js' },
  { src: '~/plugins/client-only.js', mode: 'client' }, // 클라이언트 사이드에서만
  { src: '~/plugins/server-only.js', mode: 'server' } // 서버 사이드에서만
```

```json
    "baseURL": "/",
    "buildAssetsDir": "/_nuxt/",
    "cdnURL": ""
  }
}
```

```js
export default {
```
